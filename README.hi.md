## यह क्या है?

एस्केप द वैली एक ओरेगॉन ट्रेल-शैली का सर्वाइवल गेम है जो आपके टर्मिनल में चलता है। यात्रियों के एक समूह को प्रक्रियात्मक रूप से उत्पन्न जंगल से ले जाएं। घटनाओं, खतरों और कठिन विकल्पों को नेविगेट करते समय भोजन, पानी, गाड़ी की स्थिति और मनोबल का प्रबंधन करें।

एक वैकल्पिक एआई गेम मास्टर (ओलामा द्वारा संचालित) तीन अलग-अलग कहानी कहने वाली आवाजों के साथ आपकी यात्रा का वर्णन करता है। एक वैकल्पिक एक्सआरपीएल टेस्टनेट लेजर बैकपैक आपके आपूर्ति परिवर्तनों को ऑन-चेन रसीदों के रूप में ट्रैक करता है - यह प्रमाण कि आप जीवित रहे, या यह प्रमाण कि आपने कोशिश की।

## 1.1.0 में नया क्या है

- **स्ट्रीमिंग कथन** - जीएम टोकन-दर-टोकन लिखता है, प्रत्येक क्षण को लाइव बनाता है बजाय एक विराम के बाद एक तैयार ब्लॉक छोड़ने के।
- **ग्रेड किए गए अंत** - रन एक ग्रेड वाले उपसंहार (विजयी, अनुभवी, पिरहिक या खोया हुआ) के साथ समाप्त होते हैं जो यह बताते हैं कि कौन जीवित रहा, इसमें कितना समय लगा और यात्रा की कीमत क्या थी - न कि मृत्यु का एक पंक्ति का कारण।
- **वास्तविक दांव** - अब घटनाएं पार्टी को घायल या मार सकती हैं। एक गलत विकल्प जीवन की कीमत पर पड़ सकता है, और मृत्यु का श्रेय इसके वास्तविक कारण को दिया जाता है।
- **ऑन-लेजर समाधान प्रमाण** - एक ऑडिट मोड जो रन के निपटान रसीदों को फिर से चलाता है और उन्हें एक्सआरपीएल टेस्टनेट के विरुद्ध सत्यापित करता है, ताकि आपूर्ति इतिहास को स्वतंत्र रूप से जांचा जा सके।
- **रन कलाकृतियाँ** - प्रत्येक पूर्ण रन एक स्मृति चिन्ह छोड़ देता है: एक एक्सआरपीएल पोस्टकार्ड, आपके आँकड़े और एक निर्यात/साझा पथ।

## त्वरित शुरुआत

```bash
pip install escape-the-valley

# Or, zero-prerequisite (no Python setup) via the npm launcher — downloads a
# verified binary and runs it:
#   npx @mcptoolshop/escape-the-valley tui --seed 42

# Launch the full-screen TUI (recommended)
trail tui --seed 42

# Resume a saved game
trail tui --continue

# With AI narration (requires Ollama running locally)
trail tui --seed 42 --voice

# Spoken voice narration needs the voice extra:
#   pip install "escape-the-valley[voice]"

# With voice pacing control
trail tui --seed 42 --voice --voice-pace slow

# Without AI narration (deterministic mode)
trail tui --seed 42 --gm-off

# Use a specific Ollama model
trail tui --seed 42 --model mistral
```

## कैसे खेलें

प्रत्येक मोड़ पर आप शिविर से एक क्रिया चुनते हैं:

| क्रिया | यह क्या करता है |
|--------|-------------|
| **Travel** | घाटी के निकास की ओर बढ़ें। भोजन और पानी की लागत आती है। टूटने और घटनाओं का जोखिम। |
| **Rest** | पार्टी को ठीक करें, मनोबल को फिर से प्राप्त करें। आपूर्ति की कीमत लेकिन कोई प्रगति नहीं। |
| **Hunt** | भोजन के लिए एक मौका पाने के लिए गोला-बारूद खर्च करें। जंगलों और मैदानों में बेहतर। |
| **Repair** | गाड़ी को ठीक करने के लिए एक अतिरिक्त भाग खर्च करें। जीवित रहने के लिए महत्वपूर्ण। |

**घटनाएँ** (ए/बी/सी) विकल्पों के साथ यात्रा में बाधा डालती हैं। सतर्क विकल्प सुरक्षित होते हैं लेकिन समय की कीमत पर आते हैं। साहसिक विकल्प तेज़ होते हैं लेकिन जोखिम भरे होते हैं। कोई हमेशा सही उत्तर नहीं होता है।

**गाड़ी सब कुछ है।** यदि यह बिना किसी हिस्से के टूट जाती है, तो रन समाप्त हो जाता है। इसे आधी स्थिति से ऊपर रखें और अस्थायी ब्रेकडाउन प्रतिरोध के लिए रखरखाव विंडो (विश्राम फिर मरम्मत) करें।

**गति** गति बनाम सुरक्षा को नियंत्रित करती है। स्थिर डिफ़ॉल्ट है। कठिन गति अधिक दूरी तय करती है लेकिन अधिक आपूर्ति जलाती है और गाड़ियों को तेजी से तोड़ती है।

**बचाव वाल्व** (कठिन राशन, हताश मरम्मत, कार्गो छोड़ना) आपात स्थिति के लिए मौजूद हैं। उनके दुष्प्रभाव और कूलडाउन होते हैं - अंतिम उपाय, रणनीतियाँ नहीं।

अधिक गहन युक्तियों के लिए, [सर्वाइवल गाइड](https://mcp-tool-shop-org.github.io/escape-the-valley/handbook/survival-guide/) देखें।

## जीएम प्रोफाइल

एआई कथाकार यांत्रिकी को नहीं, बल्कि स्वर को आकार देता है। तीनों प्रोफाइल एक ही गेम खेलते हैं।

- **क्रॉनिकलर** - जमीनी, व्यावहारिक, संक्षिप्त। न्यूनतम लोककथाएँ। जो हुआ उसका वर्णन करता है।
- **फायरसाइड** - गंभीर अलाव कथाकार। सूक्ष्म अलौकिक क्षण। डिफ़ॉल्ट।
- **लैंटर्न-बेयरर** - अलौकिक और सीमांत, लेकिन फिर भी परिणामों में निहित। अजीब वाला।

`--gm-profile` के साथ सेट करें: `trail tui --gm-profile lantern`

## आपूर्ति

गेम दो श्रेणियों में 12 प्रकार के संसाधनों को ट्रैक करता है:

**खपत योग्य:** भोजन, पानी, जलाऊ लकड़ी, दवाएं, नमक, गोला-बारूद, लालटेन का तेल, कपड़ा

**उपकरण:** भाग, रस्सी, उपकरण, जूते

5 मुख्य आपूर्ति (भोजन, पानी, दवाएं, गोला-बारूद, भाग) सबसे महत्वपूर्ण हैं। विस्तारित आपूर्तियाँ जैसे कि जलाऊ लकड़ी, नमक, लालटेन का तेल और कपड़ा गहराई जोड़ती हैं: जलाऊ लकड़ी रात के शिविरों को ईंधन देती है, नमक भोजन की खराबी को रोकता है, लालटेन का तेल सुरक्षित रात की यात्रा को सक्षम बनाता है, और कपड़ा उपकरणों और गाड़ी के आवरण को पैच करता है।

## लेजर बैकपैक (वैकल्पिक)

लेजर बैकपैक आपके 5 मुख्य आपूर्ति (भोजन, पानी, दवाएं, गोला-बारूद, भाग) को एक्सआरपीएल टेस्टनेट पर टोकन के रूप में ट्रैक करता है। प्रत्येक शहर चेकपॉइंट ऑन-चेन निपटान रसीद रिकॉर्ड करता है। आपकी रन के अंत में, आपके ट्रेल लेजर में लेनदेन आईडी शामिल हैं जिन्हें कोई भी सत्यापित कर सकता है।

पूरी तरह से वैकल्पिक। गेम इसे बंद करके समान रूप से चलता है (डिफ़ॉल्ट)। इसे टीयूआई में एल मेनू से या सीएलआई के माध्यम से सक्षम करें:

```bash
trail ledger enable
trail ledger status
trail ledger reconcile  # retry failed settlements
```

इसके लिए `xrpl-py` निर्भरता के लिए `pip install -e ".[xrpl]"` की आवश्यकता होती है।

| `trail self-check` | गेम पर्यावरण स्वास्थ्य की जाँच करें |
| `trail version` | संस्करण दिखाएँ |
| `trail ledger status` | बैकपैक स्थिति दिखाएँ |

| `trail ledger disable` | एक्सआरपीएल बैकपैक अक्षम करें |
| `trail ledger settle` | मैन्युअल रूप से एक चेकपॉइंट का निपटान करें |
| `trail ledger reconcile` | असफल निपटानों को फिर से प्रयास करें |
| `trail ledger wallet` | वॉलेट विवरण दिखाएँ |
| `trail stats` | रन आँकड़े दिखाएँ ( `--json` का समर्थन करता है) |
| `trail parcel send <addr> <supply> <amount>` | किसी अन्य यात्री को आपूर्ति भेजें |
| `trail parcel list` | प्राप्त पार्सल सूचीबद्ध करें |
| `trail parcel accept <id>` | एक लंबित पार्सल स्वीकार करें |
| `trail parcel sent` | उन पार्सलों की सूची बनाएं जिन्हें आपने भेजा है |
| `trail wallet share` | व्यापार के लिए अपना वॉलेट पता प्रिंट करें |

## चेतावनी कॉलआउट

डिफ़ॉल्ट रूप से, गेम विस्तृत चेतावनियाँ दिखाता है ताकि नए खिलाड़ियों को जल्दी खतरे का पता लगाने में मदद मिल सके। अनुभवी खिलाड़ी इसे न्यूनतम मोड पर स्विच कर सकते हैं, जो केवल क्लिफ-एज चेतावनियाँ (अंतिम क्षण, गंभीर खतरे) दिखाता है:

```bash
trail tui --callouts minimal
trail new --callouts minimal
```

## समस्या निवारण

**यदि कुछ भी गलत लगता है, तो पहले `trail self-check` चलाएँ।** यह रिपोर्ट करता है कि क्या ओलामा पहुंच योग्य है, क्या आपका सहेजें लोड होता है और कौन सा मॉडल स्थापित है। तीन चीजें जो गलत हो सकती हैं:

| लक्षण | कारण | ठीक करें |
|---------|-------|-----|
| **Generic / no narration** | ओलामा चल नहीं रहा है (जीएम वैकल्पिक है और यह वापस आ जाता है, इससे कोई नुकसान नहीं होता) | ओलामा शुरू करें (`ollama serve`), या `--gm-off` के साथ निश्चित रूप से चलाएं। इसकी पुष्टि करने के लिए `trail self-check` चलाएं। |
| **लेज़र लंबित / निपटान विफल** | एक्सआरपीएल टेस्टनेट एक सार्वजनिक परीक्षण नेटवर्क है और कभी-कभी इसमें समस्याएँ आती हैं। | `trail ledger reconcile` विफल निपटानों को फिर से प्रयास करता है; जब नेटवर्क ठीक हो जाए तो इसे दोबारा चलाएं। किसी भी स्थिति में, स्थानीय रूप से आपूर्ति सही रहती है। |
| **Save won't resume** | `run.json` लिखते समय बीच में ही छोटा या दूषित हो गया। | इंजन इसे अस्वीकार करने से पहले `run.json.corrupt-<टाइमस्टैम्प>` के रूप में अलग कर देता है, इसलिए आपकी अगली सहेजी गई फ़ाइल सबूत को नष्ट नहीं करेगी। उस बैकअप से पुनर्प्राप्त करें, या किसी सीड से एक नया रन शुरू करें। |

पहला वर्णित चरण मॉडल लोड करता है और इसमें 10-30 सेकंड लग सकते हैं - यह सामान्य है, कोई रुकावट नहीं है। पूरी जानकारी: [समस्या निवारण पुस्तिका](https://mcp-tool-shop-org.github.io/escape-the-valley/handbook/troubleshooting/)।
